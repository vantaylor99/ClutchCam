The local Linux Compose stack now keeps one integrated live transcription owner
by default. The `local-linux` profile starts the media server, buffer worker,
and orchestrator path while leaving the standalone `transcription-worker` as an
explicit diagnostic profile and healthcheck target.

Key behavior:

- `transcription-worker` no longer belongs to the `local-linux` profile.
- The orchestrator receives the `TRANSCRIPTION_*`, `AUDIO_EXTRACT_*`,
  `AUDIO_INPUT_URL_PLAYER_*`, `FFMPEG_EXECUTABLE`, and
  `LIVE_TRANSCRIPTION_*` environment needed for opt-in live transcription.
- The diagnostic worker receives the same transcription/audio override surface
  so `.env` settings behave consistently when operators run it explicitly.
- `.env.example`, README, architecture notes, project status, and the local
  Linux runbook now describe orchestrator-owned live transcription separately
  from worker-owned JSONL diagnostics.
- Static Compose tests assert that the integrated profile cannot accidentally
  start duplicate transcription owners.
- Buffered OBS media-source injection remains out of scope for this ticket.
