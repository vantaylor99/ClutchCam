Added an opt-in, bounded runtime transcript text log for accepted
`TranscriptEvent` messages. This supports live Faster-Whisper evaluation by
showing what text reached the router before prefilter and AI decisions run.

The feature is disabled by default because transcripts can contain private
player speech. Set `TRANSCRIPT_LOG_TEXT_ENABLED=true` for evaluation runs, and
use `TRANSCRIPT_LOG_TEXT_MAX_CHARACTERS` to bound each logged line.

Validation:

- `python -m unittest tests.test_runtime_event_pipeline tests.test_dry_run_obs tests.test_linux_compose_stack tests.test_runtime_healthcheck_entrypoints -v`
