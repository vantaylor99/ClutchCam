Implemented `TRANSCRIPTION_SOURCE_MODE=vad-utterance` as an opt-in local voice activity detection source.

What changed:

- Added VAD configuration fields and environment examples for frame length, energy threshold, minimum speech, minimum silence, leading/trailing padding, and maximum utterance duration.
- Added `VadUtteranceAudioWindowDiscovery`, which wraps completed WAV chunk discovery, reads mono `pcm_s16le` WAV frames with Python's `wave` module, assembles bounded utterance WAV request files under `_vad/<stream_id>/`, and emits `AudioInputRef` values for the existing transcriber adapters.
- Kept `chunked` as the default source mode and as the fallback. VAD startup rejects non-WAV, non-mono, or non-`pcm_s16le` extraction settings.
- Added opt-in runtime pump behavior for VAD: provider partial events are rejected, and final events whose end timestamp is not newer than the last accepted final event for the same stream are suppressed by timestamp instead of text.
- Adjusted transcription response normalization so overlap windows still require timestamps, while VAD-style utterance windows whose `emit_from_seconds` equals their start can use duration fallback for text-only provider responses.

Validation run:

- `python -m unittest ai-stream-director.tests.test_transcription_audio_extraction ai-stream-director.tests.test_transcription_runtime ai-stream-director.tests.test_transcription_worker_entrypoint`
- `python -m unittest discover -s tests` from `ai-stream-director/`

Reviewer focus:

- Check whether the simple root-mean-square energy detector and default thresholds are conservative enough for real game audio, since tests use synthetic PCM.
- Check the long-speech split behavior. It finalizes at the configured maximum duration and starts the next active utterance on the following frame; this keeps requests bounded but does not duplicate the boundary frame.
- Check generated `_vad` file cleanup. It removes generated request files retained from the previous discovery pass once they are no longer returned.
