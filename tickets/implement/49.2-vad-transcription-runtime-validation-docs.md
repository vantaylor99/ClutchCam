Finish the VAD transcription feature by documenting the selected design, exposing operator-facing validation guidance, and adding tests that prove the runtime still makes switching decisions only from final transcript events.

Design decision: provider-level utterance finalization does not replace `TranscriptRouter`. The VAD source should feed more natural final transcript events into the router, and the router should continue assembling nearby same-stream finals into bounded trigger candidates. This preserves current duplicate-trigger protection and keeps future provider streaming sources from bypassing the local prefilter and AI director path.

The docs should describe the three layers plainly:

- Fixed chunk mode is the default and fallback: FFmpeg segments audio, every stable chunk is transcribed, optional WAV overlap can reduce missed boundary phrases, and only final events reach switching decisions.
- VAD utterance mode is optional: FFmpeg still normalizes stream audio, local voice activity detection groups speech around pauses before provider requests, and the provider response still normalizes to `TranscriptEvent`.
- Future provider streaming mode can plug into the same live source boundary if a provider emits partial/final transcript events directly. It is not part of this implementation unless a concrete provider adapter already exists in the repo.

Runtime validation should be documented as an operator comparison against fixed chunks:

- missed trigger phrases,
- duplicate triggers,
- trigger latency from speech end to accepted transcript event,
- CPU/GPU use on the app host and transcription backend,
- provider request count and request duration,
- backend cost when the provider is remote or metered.

Add lightweight counters or logs only where they are already consistent with current runtime style. Do not add a full metrics subsystem in this ticket. It is enough for the worker/live source to make mode, finalized utterance count, silence skip count, request failure count, and queue drops visible in tests or logs if the implementation ticket did not already expose them.

## Edge cases & interactions

- Partial events from any source mode must remain ignored by `LiveTranscriptQueueSink` and `process_transcript_event(...)`; this ticket should add regression tests if the previous tickets did not.
- Switching decisions should use final events and router candidates, not raw VAD windows or provider-specific segment objects.
- The standalone JSONL worker should include enough payload information to compare VAD mode against chunked mode without leaking transcript text unless transcript text logging is explicitly enabled where applicable.
- Startup errors for unsupported VAD settings should tell operators to use `TRANSCRIPTION_SOURCE_MODE=chunked` as the fallback.
- Healthcheck behavior should remain bounded and should not start FFmpeg or a VAD source just to validate basic runtime health.
- Documentation must distinguish source mode from request mode so operators do not confuse `TRANSCRIPTION_SOURCE_MODE=vad-utterance` with `TRANSCRIPTION_REQUEST_MODE=openai-compatible`.
- The latency soak harness should remain offline by default. If it gains a transcription comparison hook, it must be fixture-based and must not require Docker, FFmpeg, OBS, GPU inference, or network services.

## Tests and validation

- Runtime pipeline tests should prove final VAD-sourced transcript events can trigger the existing prefilter/AI path exactly like chunked final events.
- Regression tests should prove partial VAD/provider events are rejected from switching decisions in both the queue sink and direct runtime event handler.
- Worker entrypoint tests should prove JSONL diagnostics still flush valid transcript events and source failures in the selected mode.
- Config/docs tests, if present, should be updated for `TRANSCRIPTION_SOURCE_MODE` and VAD settings.
- Run the focused transcription worker, runtime event pipeline, config, and latency harness tests affected by the docs/validation changes; then run the full Python unit suite from `ai-stream-director/`.
